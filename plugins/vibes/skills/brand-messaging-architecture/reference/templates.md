## Message Layers Assessment Template

Template for evaluating messaging against Peep Laja's framework.

```markdown
## Message Layers Assessment: [Brand Name]

---

### Layer 1: Clarity — "What is it?"

**Current Message:**
> [How you describe what you are]

**Category Statement:**
> We are a [specific category]

**Assessment:**
- [ ] Leads with category, not benefits
- [ ] Uses simple, jargon-free language
- [ ] A stranger would understand in 5 seconds
- [ ] No buzzwords or marketing-speak

**Grade:** [Pass / Needs Work]

**If Needs Work:**
[Specific revision needed]

---

### Layer 2: Relevance — "Is it for me?"

**Current Message:**
> [How you address their pain points]

**Target Statement:**
> For [specific audience] who [specific problem/situation]

**Assessment:**
- [ ] Names the target audience specifically
- [ ] Addresses a pain they recognize
- [ ] Uses their language, not ours
- [ ] They would say "that's me!"

**Grade:** [Pass / Needs Work]

**If Needs Work:**
[Specific revision needed]

---

### Layer 3: Value — "What do I get?"

**Current Message:**
> [Benefits and outcomes you promise]

**Value Statement:**
> You get [specific benefit/outcome]

**Assessment:**
- [ ] States benefits, not features
- [ ] Quantifies where possible
- [ ] Outcome-focused, not process-focused
- [ ] Compelling and desirable

**Grade:** [Pass / Needs Work]

**If Needs Work:**
[Specific revision needed]

---

### Layer 4: Differentiation — "Why you over alternatives?"

**Current Message:**
> [Why choose you specifically]

**Differentiation Statement:**
> Unlike [alternatives], we [unique difference]

**Assessment:**
- [ ] Names alternatives (not just "competitors")
- [ ] States something ONLY we can claim
- [ ] Passes the "Only" test
- [ ] Backed by proof

**Grade:** [Pass / Needs Work]

**If Needs Work:**
[Specific revision needed]

---

### Overall Assessment

**Layer Sequence Check:**
- [ ] Clarity is established before Relevance
- [ ] Relevance is established before Value
- [ ] Value is established before Differentiation
- [ ] Each layer builds on the previous

**Final Grade:** [Strong / Adequate / Needs Work]

**Priority Fixes:**
1. [Most critical fix]
2. [Second priority]
3. [Third priority]
```

| Would they pay for this? | [Yes/Partially/No] |

[Customer research that supports/contradicts appeal]

## Output Validation Checklist

When reviewing messaging documentation, verify:

**Value Proposition**
- [ ] Core value proposition clearly stated
- [ ] Onlyness statement can be completed credibly
- [ ] Geoffrey Moore statement written
- [ ] Three benefit types identified (functional, emotional, self-expression)
- [ ] Passes MECLABS criteria

**Message Layers**
- [ ] Clarity established first
- [ ] Relevance follows clarity
- [ ] Value follows relevance
- [ ] Differentiation follows value
- [ ] Each layer assessed

**Brand Pillars**
- [ ] 3-5 pillars defined (not more)
- [ ] Each pillar has core message
- [ ] Each pillar has proof points (rational, emotional, visual)
- [ ] Pillars are distinct, not overlapping
- [ ] Pillars support the value proposition

**Messaging Hierarchy**
- [ ] Primary message identified
- [ ] Secondary messages support primary
- [ ] Tertiary messages for specific contexts
- [ ] Hierarchy is clear and actionable

**Application**
- [ ] Messages adapted by audience segment
- [ ] Messages adapted by touchpoint
- [ ] Objection-handling messages included
- [ ] Do/Don't guidelines provided

**Quality**
- [ ] Tested with real customers (or testing plan)
- [ ] Evidence backs all claims
- [ ] Free of jargon and buzzwords
- [ ] Competitors can't say the same things
