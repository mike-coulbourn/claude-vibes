## Naming Brief Template

Use these questions during discovery to understand naming requirements.

```
## Naming Brief

### Brand Foundation
1. What does your company/product do in one sentence?
2. What makes you different from competitors?
3. Where are you now (mission) and where would you like to be (vision)?

### Target Audience
4. Who is your primary customer? (Be specific, not "everyone")
5. How would you describe your ideal customer's personality?
6. What words would resonate with them?

### Competitive Landscape
7. Name your top 3-5 competitors
8. Which competitor names do you admire? Why?
9. What naming patterns do you see in your industry?

### Name Preferences
10. Where on the naming spectrum do you prefer?
    [ ] Descriptive (Pizza Hut, TripAdvisor)
    [ ] Suggestive (Netflix, Slack, Airbnb)
    [ ] Abstract (Kodak, Xerox, Google)

11. What style feels right?
    [ ] Modern/Tech
    [ ] Classic/Traditional
    [ ] Playful/Friendly
    [ ] Professional/Serious
    [ ] Bold/Edgy

12. Length preference?
    [ ] One syllable (Slack, Zoom)
    [ ] Two syllables (Google, Apple)
    [ ] Three+ syllables (Amazon, Spotify)
    [ ] No preference

### Constraints
13. What words MUST be avoided?
14. What associations should be avoided?
15. Any sounds or letters to avoid?
16. International considerations? Which markets?

### Practical Requirements
17. Is .com domain required or are alternatives acceptable?
18. Budget for premium domain acquisition? (Y/N, range)
19. Trademark classes needed?

### Previous Attempts
20. What names have already been considered?
21. Why were they rejected?
22. Are there any names you've been using informally?
```

## Output Validation Checklist

When reviewing naming output, verify:

**Discovery Quality**
- [ ] Naming brief completed or key questions answered
- [ ] Brand positioning understood
- [ ] Target audience defined
- [ ] Competitive landscape mapped
- [ ] Preferences and constraints captured

**Generation Quality**
- [ ] Multiple naming categories explored
- [ ] At least 10+ initial candidates generated
- [ ] Sound symbolism considered
- [ ] Both obvious and creative options included

**Evaluation Quality**
- [ ] Each name has SMILE evaluation
- [ ] SCRATCH filter applied
- [ ] Sound symbolism analyzed
- [ ] Strategic rationale provided

**Availability Verification**
- [ ] Domain availability verified via whois (not just lookup)
- [ ] Alternative TLDs checked if .com taken
- [ ] Social handles checked
- [ ] Basic trademark search conducted

**Presentation Quality**
- [ ] 5-7 vetted options presented
- [ ] Comparison matrix provided
- [ ] Clear recommendation with rationale
- [ ] Names only presented if domain verified available

**Selection Documentation**
- [ ] Final name documented with full analysis
- [ ] Domain status confirmed current
- [ ] Pronunciation and spelling notes included
- [ ] Trademark status noted
- [ ] Runner-up names documented
- [ ] Immediate action items listed

## Quick Reference Cards

### SMILE Quick Check
Ask for each candidate:
1. Does it suggest something about the brand?
2. Will people remember it after hearing once?
3. Does it create a mental picture?
4. Can we build a brand world around it?
5. Does it evoke emotion?

### SCRATCH Quick Filter
Reject if:
1. People will misspell it
2. It sounds like a competitor
3. It limits future growth
4. It feels forced or gimmicky
5. It's boring or generic
6. Only insiders get it
7. People will avoid saying it

### Domain Priority
1. .com (always first choice)
2. .io (tech-friendly alternative)
3. .co (viable for startups)
4. [name]app.com (product-focused)
5. get[name].com (action-oriented)
