## AI Voice Prompt Template

For AI content generation consistency.

```
### AI Voice Guidelines for [Brand Name]

---

#### Base Voice Prompt

Use this prompt when generating content:

```
Write as [Brand Name]: [Trait 1], [Trait 2], [Trait 3], [Trait 4] — but not [anti-trait 1], [anti-trait 2], [anti-trait 3].

Our voice sounds like [one-sentence description].

Example of our voice:
"[Sample sentence that demonstrates all traits]"

When writing:
- [Specific instruction 1]
- [Specific instruction 2]
- [Specific instruction 3]

Never:
- [Anti-instruction 1]
- [Anti-instruction 2]
- [Anti-instruction 3]
```

---

#### Channel-Specific Prompts

**For Marketing Content:**
```
[Base prompt] + In marketing contexts, emphasize [trait] and dial up [quality].
Be [tone description].
```

**For Support Content:**
```
[Base prompt] + In support contexts, prioritize [trait] and be especially [quality].
The reader may be [emotional state], so [response approach].
```

**For Social Media:**
```
[Base prompt] + For social media, allow more [flexibility].
Keep it [length] and [quality].
```

---

#### Human Review Checklist

Before publishing AI-generated content, verify:

- [ ] Does this pass the "airport test" — recognizable as our brand?
- [ ] Are all "this but not that" constraints respected?
- [ ] Is the tone appropriate for this specific context?
- [ ] Does vocabulary align with our guidelines?
- [ ] Would we say it this way in person?

---

#### Sample Corrections

- [Issue 2]
```

## Output Validation Checklist

When reviewing voice documentation, verify:

**Voice Foundation**
- [ ] Voice connected to purpose, values, archetype
- [ ] NNGroup four dimensions plotted
- [ ] One-sentence voice description is memorable
- [ ] Clear distinction from competitors

**Personality Traits**
- [ ] 3-5 specific traits defined
- [ ] Each trait has "this but not that" constraint
- [ ] Each trait has concrete do/don't examples
- [ ] Traits work together coherently

**Tone Adaptation**
- [ ] Channel-specific guidelines included
- [ ] Emotional state responses documented
- [ ] Tone boundaries defined
- [ ] Examples for each context

**Vocabulary**
- [ ] Words to use listed with rationale
- [ ] Words to avoid with alternatives
- [ ] Industry jargon policy defined
- [ ] Audience language incorporated

**Examples**
- [ ] Before/after examples included
- [ ] Multiple content types covered
- [ ] Examples are concrete and usable
- [ ] AI guidelines included

**Usability**
- [ ] Summary card for quick reference
- [ ] Guidelines are actionable not vague
- [ ] Someone unfamiliar could write on-brand
- [ ] Airport test addressed
